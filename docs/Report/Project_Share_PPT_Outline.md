# 个性化旅游推荐系统开发经验分享

## PPT大纲

### 1. 开场

- **标题**：个性化旅游推荐系统开发经验分享
- **副标题**：AI辅助开发方法论探索
- **演讲人**：陈逸
- **时间**：2026年4月

### 2. AI运用方法论总览

- **核心方法**：
  1. **技能创建**：通过Skill Creator创建专用技能
  2. **多角色协作**：让AI扮演不同的项目角色
  3. **交叉评审**：让装备不同技能的AI互相评判工作
  4. **流程规范**：建立完善的AI协作流程
- **专用技能**：
  - 需求分析技能
  - 架构设计技能
  - 代码实现技能
  - 测试验证技能
- **AI角色**：
  - 项目经理
  - 架构师
  - 开发工程师
  - 测试工程师

### 3. 需求分析阶段的AI应用

- **AI角色**：装备需求分析技能的项目经理
- **AI任务**：
  - 识别核心功能点（旅游推荐、路线规划、设施查询等）
  - 按旅游前/中/后分类功能
  - 明确非功能需求（响应时间、安全性、可用性）
  - 生成需求文档初稿
- **成果**：需求规格说明书

### 4. 技术栈选择阶段的AI应用

- **AI角色**：装备架构设计技能的架构师
- **AI任务**：
  - 基于课程约束评估技术选项
  - 考虑前后端分离架构可行性
  - 评估各技术的生态和学习曲线
  - 推荐技术栈
- **成果**：技术栈方案（Vue 3 + Spring Boot 3.2）

### 5. 开发计划与架构设计的AI应用

- **AI角色**：
  - 装备架构设计技能的架构师
  - 装备项目管理技能的项目经理
- **AI任务**：
  - 架构师设计系统架构和模块划分
  - 项目经理制定开发计划和里程碑
- **交叉评审**：
  - 架构师评估项目经理提出的需求是否清晰可执行
  - 项目经理根据反馈迭代需求文档
- **AI协作流程**：
  - 开发前：阅读PROJECT\_CONTEXT.md了解项目状态
  - 开发后：更新HANDOFF.md记录进度
  - 确保工作进度在不同成员和AI会话之间同步

### 6. 项目初始化与基础架构的AI应用

- **AI角色**：
  - 装备代码实现技能的开发工程师
  - 装备测试验证技能的测试工程师
- **AI任务**：
  - 开发工程师生成项目基础架构代码
  - 测试工程师自动编写测试用例
- **交叉评审**：
  - 架构师审查开发工程师的代码是否符合设计要求
  - 产品经理记录项目进度
- **成果**：项目结构搭建、功能开发和测试

### 7. 数据采集与处理的AI应用

- **AI角色**：装备代码实现技能的开发工程师
- **AI任务**：
  - 生成OSM数据采集py脚本
  - 集成OpenStreetMap API抓取真实地图数据
  - 处理和转换地图数据格式
  - 生成道路网络和POI数据
- **优势**：
  - 节省手动编写脚本的时间
  - 提高数据处理的准确性

### 8. 方法论总结与收获

- **方法论总结**：
  1. **技能创建**：通过Skill Creator创建专用技能
  2. **多角色协作**：让AI扮演不同的项目角色
  3. **迭代优化**：让不同角色的AI互相评判工作
  4. **流程规范**：建立完善的AI协作流程
- **收获**：
  - 提高开发效率
  - 保证代码质量
  - 确保项目一致性
  - 掌握AI辅助开发方法
