# 个性化旅游推荐系统开发经验分享

## 演讲稿

### 开场（15秒）

各位老师、同学们，大家好！我是陈逸，来自数据结构课程项目组。今天我想和大家分享我们团队如何在项目开发中运用AI工具提升效率，以及我们探索出的AI辅助开发方法论。

### AI运用方法论总览（20秒）

在开始分享具体开发流程之前，我想先介绍我们的核心AI运用方法论：

1. **技能创建**：通过Skill Creator创建专用技能，如需求分析、架构设计、代码实现和测试验证技能
2. **多角色协作**：让AI扮演不同的项目角色，如产品经理、架构师、开发工程师、测试工程师和项目经理
3. **交叉评审**：让装备不同技能的AI互相评判对方的工作，实现迭代优化
4. **流程规范**：建立完善的AI协作流程，确保开发的一致性和质量

<br />

### 需求分析阶段的AI应用（30秒）

在需求分析阶段，我们唤醒agent的需求分析技能，然后让装备了该技能的AI扮演产品经理角色。这个AI产品经理帮助我们：

1. 识别核心功能点，如旅游推荐、路线规划、设施查询等
2. 按照旅游前、旅游中和旅游后进行功能分类
3. 明确非功能需求，如响应时间、安全性和可用性
4. 生成需求文档初稿

### 技术栈选择阶段的AI应用（30秒）

在技术栈选择阶段，我们调用架构师技能，并让装备了该技能的AI扮演架构师角色。这个AI架构师：

1. 基于课程约束（如自主实现核心算法、禁用Java内置集合）评估技术选项
2. 考虑前后端分离架构的可行性
3. 评估各技术的生态和学习曲线
4. 最终推荐了Vue 3 + Spring Boot 3.2的技术栈

### 开发计划与架构设计的AI应用（30秒）

在制定开发计划和架构设计时，我们：

1. AI架构师设计系统架构和模块划分
2. 让AI项目经理制定开发计划和里程碑
3. 同时让不同角色的AI互相审查对方的工作：
   - 架构师评估项目经理提出的需求是否清晰可执行
   - 项目经理根据反馈迭代需求文档

我们建立了完善的AI协作流程：每次开发前让AI阅读PROJECT\_CONTEXT.md了解项目状态，开发后更新HANDOFF.md记录进度，方便工作进度在不同成员以及不同AI会话之间同步。

### 项目初始化与基础架构的AI应用（30秒）

在项目初始化和基础架构搭建阶段，我们：

1. 调用代码实现技能，让AI开发工程师依照需求文档和技术文档的指导生成项目基础架构代码
2. 并且使用测试验证技能，让AI测试工程师自动编写测试用例
3. 完成一轮开发后让AI架构师审查开发工程师的代码是否符合设计要求，并且让AI产品经理记录项目进度

通过这种方式，我们快速完成了项目结构搭建与功能开发和测试。

### 数据采集与处理的AI应用（20秒）

在数据采集阶段，我们让AI开发工程师生成OSM数据采集py脚本，帮助我们：

1. 集成OpenStreetMap API抓取真实地图数据
2. 处理和转换地图数据格式
3. 生成道路网络和POI数据

这大大节省了我们手动编写脚本的时间，同时提高了数据处理的准确性。

### 方法论总结与收获（20秒）

通过这次项目，我们探索出了一套有效的AI辅助开发方法论：

1. **技能创建**：通过Skill Creator创建专用技能
2. **多角色协作**：让AI扮演不同的项目角色
3. **迭代优化**：让不同角色的AI互相评判对方的工作
4. **流程规范**：建立完善的AI协作流程
